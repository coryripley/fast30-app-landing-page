You’re fighting two things in the Replit version:
On mobile you’ve set .hero-device to height: auto, so the hero’s height grows to whatever the PNG’s rendered height becomes. That’s why the hand/phone balloons and “pushes into” the next section.
The phone image sizing is effectively controlled by the #hero-phone-final { … !important } rule, so changing the class (.heroPhoneImg) often appears to do nothing.
The clean fix is to bring the hero phone block back to the exact Figma Make layout behavior: fixed hero height on mobile (70vh), and the phone positioned absolutely inside that box, with a height cap.
Fix (recommended): restore the Figma Make hero-phone implementation

id="hero-phone-final" and className="heroPhoneImg"

<div className="flex justify-center items-center hero-phone-wrapper" style={{ zIndex: 10, position: 'relative' }}>

    src={mediaDeviceHandHoldingPhone}
    alt="FAST30 App Dashboard"

2) Fix the real cause: mobile hero height is currently “auto”
In the <style dangerouslySetInnerHTML> for .hero-device, change the mobile rule from your current height: auto/min-height: 70vh setup to a fixed height like Figma:
Find this in your Replit code:
@media (max-width: 768px) {
  .hero-device { 
    width: 100vw !important;
    max-width: 100vw !important;
    height: auto !important;
    min-height: 70vh !important;
    /* ... */
  }
}
Replace it with:
@media (max-width: 768px) {
  .hero-device { 
    width: 100% !important;
    height: 70vh !important;
    position: relative !important;
  }
}
That one change is what prevents the phone asset from dictating the hero’s height (and blowing up like your first screenshot).
If you want to keep <img> (alternate)
You can keep the <img> and just change the mobile rule to constrain by height (not width):
@media (max-width: 768px) {
  .hero-phone-wrapper { position: absolute !important; inset: 0 !important; display: block !important; }
  #hero-phone-final {
    height: calc(70vh - 25px) !important;
    width: auto !important;
    max-height: calc(70vh - 25px) !important;
    position: absolute !important;
    left: 50% !important;
    top: 2.5px !important;
    transform: translateX(-50%) !important;
  }
}
Why your resizing “did nothing”
Because this exists in your Replit code:
#hero-phone-final { ... !important; max-height: unset !important; }
So edits to .heroPhoneImg won’t win. Once you switch to the fixed 70vh hero and absolute phone sizing, the phone will snap back to the intended scale (your second screenshot).
